# 사용자 경험 (UX) 설명

## 👤 직원 (Employee) 관점

### 📱 매장 도착 - 출근하기

**상황**: 알바생이 매장에 도착했습니다.

**화면**:
```
┌─────────────────────────┐
│   PartTimeHR            │
│                         │
│   [출근하기]            │
│   (큰 버튼, 한 번 클릭) │
└─────────────────────────┘
```

**동작**:
1. 직원이 "출근하기" 버튼을 한 번 클릭
2. 즉시 현재 시간이 기록됨 (예: 2025-12-25 09:00)
3. 화면이 바뀜

**결과 화면**:
```
┌─────────────────────────┐
│   출근 완료!            │
│   오늘: 2025-12-25      │
│   출근 시간: 09:00      │
│                         │
│   [휴게시작]  [퇴근하기]│
└─────────────────────────┘
```

**느낌**: 
- ✅ **간단함**: 버튼 한 번 클릭으로 끝
- ✅ **빠름**: 즉시 기록됨
- ✅ **확실함**: 출근 시간이 바로 보임

---

### 🍽️ 점심시간 - 휴게하기

**상황**: 점심시간이 되어서 휴게를 시작합니다.

**화면**:
```
┌─────────────────────────┐
│   근무 중               │
│   출근: 09:00           │
│                         │
│   [휴게시작]  [퇴근하기]│
└─────────────────────────┘
```

**동작**:
1. "휴게시작" 버튼 클릭
2. 즉시 휴게 시작 시간 기록 (예: 13:00)
3. 화면 업데이트

**결과 화면**:
```
┌─────────────────────────┐
│   휴게 중               │
│   출근: 09:00           │
│   휴게 시작: 13:00      │
│                         │
│   [휴게끝]              │
└─────────────────────────┘
```

**느낌**:
- ✅ **편리함**: 휴게 시작도 버튼 한 번
- ✅ **자동화**: 시간 자동 기록, 계산도 자동

---

### 🍜 점심 끝 - 휴게 종료

**상황**: 점심을 먹고 다시 일을 시작합니다.

**동작**:
1. "휴게끝" 버튼 클릭
2. 즉시 휴게 종료 시간 기록 (예: 14:00)
3. 화면 업데이트

**결과 화면**:
```
┌─────────────────────────┐
│   근무 중               │
│   출근: 09:00           │
│   휴게: 13:00 ~ 14:00   │
│                         │
│   [휴게시작]  [퇴근하기]│
└─────────────────────────┘
```

**느낌**:
- ✅ **유연함**: 휴게가 없어도 되고, 여러 번 해도 됨 (설계상)
- ✅ **명확함**: 휴게 시간이 표시됨

---

### 🏠 하루 일 끝 - 퇴근하기

**상황**: 하루 일이 끝나서 퇴근합니다.

**동작**:
1. "퇴근하기" 버튼 클릭
2. 즉시 퇴근 시간 기록 (예: 18:00)
3. 근무 시간 자동 계산
4. 완료 화면 표시

**결과 화면**:
```
┌─────────────────────────┐
│   오늘 근무 완료!       │
│                         │
│   출근: 09:00           │
│   휴게: 13:00 ~ 14:00   │
│   퇴근: 18:00           │
│                         │
│   총 근무: 9시간        │
│   휴게: 1시간           │
│   실제 근무: 8시간      │
│                         │
│   [확인]                │
└─────────────────────────┘
```

**느낌**:
- ✅ **만족감**: 하루 일 끝, 근무 시간이 바로 보임
- ✅ **명확함**: 총 근무 시간, 휴게 시간, 실제 근무 시간이 자동 계산됨
- ✅ **간편함**: 계산할 필요 없음

---

### 📊 내 근무 기록 보기

**상황**: 이번 달에 얼마나 일했는지 확인하고 싶습니다.

**화면**:
```
┌─────────────────────────┐
│   내 근무 기록          │
│                         │
│   12/25 (월)            │
│   09:00 ~ 18:00         │
│   실제 근무: 8시간      │
│   ─────────────────     │
│   12/24 (일)            │
│   10:00 ~ 19:00         │
│   실제 근무: 9시간      │
│   ─────────────────     │
│   총 근무: 17시간       │
└─────────────────────────┘
```

**느낌**:
- ✅ **한눈에 보임**: 날짜별로 정리되어 있음
- ✅ **총계 표시**: 총 근무 시간도 자동 계산

---

## 👔 고용주 (Employer) 관점

### 📋 직원 출근 기록 확인

**상황**: 사장님이 오늘 누가 출근했는지 확인합니다.

**화면**:
```
┌─────────────────────────┐
│   출근 기록 관리        │
│                         │
│   [전체] [오늘] [이번주]│
│   [직원 선택 ▼]         │
│                         │
│   김직원 - 12/25        │
│   09:00 출근            │
│   13:00 휴게시작        │
│   14:00 휴게끝          │
│   18:00 퇴근            │
│   실제 근무: 8시간      │
│   [수정] [삭제]         │
│   ─────────────────     │
│   이직원 - 12/25        │
│   10:00 출근            │
│   근무 중...            │
│   [수정] [삭제]         │
└─────────────────────────┘
```

**느낌**:
- ✅ **한눈에 파악**: 모든 직원의 출근 상태가 보임
- ✅ **실시간**: 누가 지금 근무 중인지 바로 알 수 있음

---

### ✏️ 잘못 입력된 시간 수정하기

**상황**: 직원이 실수로 출근 시간을 잘못 눌렀습니다. (09:00인데 10:00으로 기록됨)

**동작**:
1. 해당 기록의 "수정" 버튼 클릭
2. 시간 입력 화면 표시

**화면**:
```
┌─────────────────────────┐
│   출근 기록 수정        │
│                         │
│   출근 시간:            │
│   [10:00] → [09:00]     │
│                         │
│   휴게 시작:            │
│   [13:00]               │
│                         │
│   휴게 끝:              │
│   [14:00]               │
│                         │
│   퇴근 시간:            │
│   [18:00]               │
│                         │
│   [저장]  [취소]        │
└─────────────────────────┘
```

**결과**:
- 출근 시간이 09:00으로 수정됨
- 근무 시간이 자동으로 재계산됨 (9시간 → 10시간)

**느낌**:
- ✅ **자유로움**: 언제든지 수정 가능
- ✅ **편리함**: 부분 수정도 가능 (출근 시간만 수정해도 됨)
- ✅ **자동 계산**: 수정하면 근무 시간이 자동으로 다시 계산됨

---

### ➕ 수동으로 출근 기록 등록하기

**상황**: 직원이 출근 버튼을 누르는 걸 깜빡했거나, 과거 날짜 기록을 추가해야 합니다.

**동작**:
1. "수동 등록" 버튼 클릭
2. 직원 선택, 날짜, 시간 입력

**화면**:
```
┌─────────────────────────┐
│   출근 기록 수동 등록   │
│                         │
│   직원: [김직원 ▼]      │
│   날짜: [2025-12-25]    │
│                         │
│   출근 시간:            │
│   [09:00]               │
│                         │
│   휴게 시작: (선택)     │
│   [13:00]               │
│                         │
│   휴게 끝: (선택)       │
│   [14:00]               │
│                         │
│   퇴근 시간:            │
│   [18:00]               │
│                         │
│   [등록]  [취소]        │
└─────────────────────────┘
```

**느낌**:
- ✅ **유연함**: 언제든지 추가 가능
- ✅ **완전한 제어**: 모든 시간을 직접 입력 가능

---

### 🗑️ 잘못된 기록 삭제하기

**상황**: 테스트로 만든 기록이나 완전히 잘못된 기록을 삭제해야 합니다.

**동작**:
1. 해당 기록의 "삭제" 버튼 클릭
2. 확인 메시지 표시
3. 삭제 완료

**느낌**:
- ✅ **간단함**: 버튼 한 번으로 삭제
- ✅ **자유로움**: 언제든지 삭제 가능 (완료된 기록도 삭제 가능)

---

### 📊 월별 근무 시간 통계 보기

**상황**: 이번 달에 직원들이 얼마나 일했는지 확인합니다.

**화면**:
```
┌─────────────────────────┐
│   12월 근무 통계        │
│                         │
│   김직원                │
│   총 근무: 120시간      │
│   일한 날: 15일         │
│   평균: 8시간/일        │
│   ─────────────────     │
│   이직원                │
│   총 근무: 96시간       │
│   일한 날: 12일         │
│   평균: 8시간/일        │
└─────────────────────────┘
```

**느낌**:
- ✅ **명확함**: 한눈에 통계가 보임
- ✅ **편리함**: 급여 계산에 바로 사용 가능

---

## 🎯 전체적인 사용자 경험 요약

### 직원의 느낌
1. **간단함**: 버튼만 누르면 끝
   - 출근, 휴게, 퇴근 모두 버튼 한 번
   - 복잡한 입력 없음

2. **빠름**: 즉시 반영
   - 클릭하면 바로 기록됨
   - 대기 시간 없음

3. **확실함**: 내가 한 일이 바로 보임
   - 출근 시간, 근무 시간이 바로 표시
   - 계산할 필요 없음

4. **부담 없음**: 실수해도 괜찮음
   - 사장님이 수정해줄 수 있음
   - 걱정할 필요 없음

---

### 고용주의 느낌
1. **자유로움**: 언제든지 수정/삭제 가능
   - 완료된 기록도 수정 가능
   - 잘못된 기록 자유롭게 삭제

2. **편리함**: 모든 기록을 한눈에
   - 직원별, 날짜별 필터링
   - 실시간 상태 확인

3. **정확함**: 자동 계산
   - 근무 시간 자동 계산
   - 휴게 시간 자동 제외

4. **완전한 제어**: 수동 등록도 가능
   - 과거 날짜 기록 추가
   - 빠뜨린 기록 보완

---

## 💡 핵심 가치

### "복잡하지 않고 간단하고 빠른"

- ✅ **직원**: 버튼만 누르면 끝
- ✅ **고용주**: 자유롭게 관리 가능
- ✅ **자동화**: 계산은 모두 자동
- ✅ **유연함**: 언제든지 수정 가능

**결과**: 사용자가 시스템에 신경 쓸 필요 없이, 일에만 집중할 수 있음! 🎯
